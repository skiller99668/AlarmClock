# AlarmClock

A personal project designed to make getting up in the mornings easier and prevent snoozing past your alarm. It's a custom ESP32-based alarm clock with an AI-generated math wake-up challenge, LED sunrise simulation, and audio playback.

## Features

- **Math problem alarm** — the alarm won't stop until you solve a math problem on the keypad. Problems are generated by a hosted LLM (Groq) over Wi-Fi, with a local generator as fallback when offline
- **Adaptive difficulty** — the problem difficulty (scale of 1–55) automatically rises or falls based on how quickly you solved the previous one
- **LED sunrise simulation** — 15 minutes before the alarm, a WS2812B LED strip gradually brightens a warm amber glow from off to full to simulate a natural sunrise
- **LCD display** — shows the current time, alarm time, and math problem
- **4x4 keypad input** — used to set the time, set the alarm, and answer the math problem
- **Audio** — passive buzzer for alarm sound, with DFPlayer Mini + MP3 playback planned (library wired up, playback not yet enabled)
- **RTC timekeeping** — DS1302 module with coin-cell backup keeps time even when unplugged
- **Persistent alarm** — the alarm time survives a reboot/power loss (stored in NVS via Preferences)

## Hardware

| Component | Part |
|---|---|
| Microcontroller | ESP32 DevKit (esp32doit-devkit-v1) |
| Display | LCD1602 with I2C adapter |
| Input | 4x4 membrane keypad |
| Lighting | WS2812B LED strip |
| Audio | Passive buzzer (DFPlayer Mini + speaker planned) |
| Timekeeping | DS1302 RTC module |
| Power | USB 5V |

## Wiring

| Component | ESP32 Pin |
|---|---|
| LCD SDA | GPIO 21 |
| LCD SCL | GPIO 22 |
| RTC DAT (IO) | GPIO 27 |
| RTC SCLK | GPIO 14 |
| RTC CE (RST) | GPIO 26 |
| Keypad R1-R4 | GPIO 19, 18, 5, 17 |
| Keypad C1-C4 | GPIO 16, 4, 0, 2 |
| Buzzer | GPIO 12 |
| LED strip DIN | GPIO 15 |

The LCD is the only I2C device (address `0x27` on SDA/SCL). The DS1302 RTC uses a separate 3-wire interface (DAT / SCLK / CE), **not** I2C.

### Configuration

The math problems are generated by a hosted, OpenAI-compatible LLM (Groq by default), so the clock needs Wi-Fi and an API key. Create `src/config.h`:

```cpp
#define WIFI_SSID     "your-network"
#define WIFI_PASSWORD "your-password"

// Hosted OpenAI-compatible LLM (Groq by default).
// Get a free key at https://console.groq.com/keys
#define LLM_URL   "https://api.groq.com/openai/v1/chat/completions"
#define LLM_MODEL "openai/gpt-oss-120b"
#define LLM_API_KEY "gsk_..."
```

This file holds secrets — keep it out of version control (add `src/config.h` to `.gitignore`). If there's no network or the request fails, the clock falls back to a locally generated arithmetic problem.

## Usage

### Setting the time
On boot, press `D` to enter time setting mode. Enter the hour (0-23) followed by `#`, then the minute followed by `#`.

### Setting the alarm
Press `A` to enter alarm setting mode. Enter the hour followed by `#`, then the minute followed by `#`. Press `A` again at any time to cancel an active alarm.

### Checking the alarm
Press `B` to see when the alarm is set, or whether one is set at all.

### Alarm sequence
1. 15 minutes before the alarm, the LED strip begins brightening
2. At the alarm time, the buzzer sounds and a math problem appears on the LCD
3. Solve the problem using the keypad and press `#` to submit
4. Once correct, press `*` to silence and dismiss the alarm

### Keypad layout
```
1  2  3  A
4  5  6  B
7  8  9  C
*  0  #  D
```

In Clock Mode:
- `A` — Set alarm / Cancel alarm
- `B` — View set alarm time
- `D` — Set new time

In Alarm/Time Setting Mode:
- `#` — Confirm / Submit
- `A` — Backspace
- `D` — Cancel

When Alarm Sounds (Equation Mode):
- `#` — Submit answer
- `A` — Backspace
- `*` — Stop alarm (only after the answer is correct)
